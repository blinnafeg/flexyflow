# Инструкция по подключению к удаленному репозиторию GitHub

## Шаг 1: Создание репозитория на GitHub

1. Перейдите на сайт https://github.com и войдите в свой аккаунт
2. Нажмите кнопку "New repository" (Новый репозиторий)
3. Введите имя репозитория: `flexyflow`
4. Убедитесь, что опция "Initialize this repository with a README" НЕ отмечена
5. Нажмите кнопку "Create repository"

## Шаг 2: Подключение локального репозитория к удаленному

После создания репозитория на GitHub, выполните следующие команды в терминале в корне проекта:

```bash
# Добавление удаленного репозитория (замените YOUR_USERNAME на ваше имя пользователя GitHub)
git remote add origin https://github.com/YOUR_USERNAME/flexyflow.git

# Проверка подключения
git remote -v

# Отправка изменений в удаленный репозиторий
git branch -M main
git push -u origin main
```

## Шаг 3: Проверка результата

После выполнения команд, перейдите на страницу вашего репозитория на GitHub. Вы должны увидеть все файлы проекта, включая:

- Исходный код приложения
- Файлы конфигурации
- Документацию (PASSPORT.md)
- Схему базы данных

## Шаг 4: Регулярное резервное копирование

Для регулярного резервного копирования проекта используйте следующие команды:

```bash
# Добавление изменений в репозиторий
git add .

# Создание коммита с осмысленным сообщением
git commit -m "Описание внесенных изменений"

# Отправка изменений в удаленный репозиторий
git push origin main
```

## Шаг 5: Создание резервной копии локально

Для создания локальной резервной копии проекта выполните:

```bash
# Создание архива репозитория
git archive --format=zip --output=flexyflow-backup.zip main
```

Это создаст файл `flexyflow-backup.zip` с полной копией проекта на момент последнего коммита.

## Дополнительные рекомендации

1. Используйте теги для обозначения важных версий:
   ```bash
   git tag -a v1.0.0 -m "Версия 1.0.0"
   git push origin v1.0.0
   ```

2. Создайте ветки для разработки новых функций:
   ```bash
   git checkout -b feature/new-feature
   # После завершения разработки
   git checkout main
   git merge feature/new-feature
   git push origin main
   ```

3. Регулярно синхронизируйте локальный репозиторий с удаленным:
   ```bash
   git pull origin main
   ```

Следуя этим инструкциям, вы сможете успешно подключить ваш локальный репозиторий к GitHub и обеспечить регулярное резервное копирование проекта FlexyFlow.
